# 快速开始（OceanBase 后端）

本文说明如何用 OceanBase 作为 Connectors 后端：**以 whl 包安装为主**，使用 `connectors` 与 `sync-ob` 命令完成配置、创建 connector、启动任务与一次性同步。

---

## 一、前置条件

1. **OceanBase 实例**（MySQL 协议，如 4.x/5.x）
   - 已创建好目标库（如 `test`），CLI 会在该库下自动创建系统表与内容表。
2. **安装 Connectors 包**：通过 whl 或从源码安装（见第二节）。OceanBase 后端依赖 `aiomysql`，会随包安装。

---

## 二、安装（whl 包方式）

依赖 `elasticsearch-connectors-sdk` 未发布到 PyPI，在本仓库中为 `libs/connectors_sdk`。**先装 SDK，再装 whl**。

在**仓库根目录**（与 `app`、`libs` 同级）下执行：

```bash
# 1. 安装本地 SDK（只需执行一次）
pip install ./libs/connectors_sdk

# 2. 安装 whl，或从源码安装
pip install ./app/connectors_service/dist/elasticsearch_connectors-*.whl
```

若尚未打包 whl，可改为从源码安装：

```bash
pip install ./libs/connectors_sdk
pip install ./app/connectors_service
```

安装完成后，在任意目录可使用命令：`connectors`、`sync-ob`。

### 2.1 打包 whl（可选）

若需自行打包 whl，在 `app/connectors_service` 下执行：

```bash
pip install build
python -m build --wheel
```

生成的 whl 位于 `dist/`，例如 `dist/elasticsearch_connectors-9.4.0-py3-none-any.whl`。

## 三、配置文件

### 3.1 配置结构

CLI 通过 `backend: oceanbase` + `oceanbase` 段识别 OceanBase：

```yaml
backend: oceanbase
oceanbase:
  host: "127.0.0.1"
  port: 2881
  user: "root@test"
  password: "your_password"
  db_name: "test"
```

| 项 | 说明 |
|----|------|
| host / port | OceanBase 地址与端口（MySQL 协议） |
| user / password | 登录账号与密码（可带租户，如 `root@test`） |
| db_name 或 database | 使用的库名，系统表与内容表建在该库下 |

### 3.2 指定配置文件（`-c`）

使用 OceanBase 时建议单独写一份 YAML，通过 `-c` 传入。**`-c` 的路径是相对当前工作目录的**；安装 whl 后可在任意目录执行，建议用绝对路径或少用 `cd` 避免找错文件。

- 在**配置所在目录**下执行时可直接写文件名：`connectors -c test-ob-config.yml ...`
- 在**仓库根目录**执行且配置在 `app/connectors_service/` 时，需写相对路径：`connectors -c app/connectors_service/test-ob-config.yml ...`
- 或先 `cd app/connectors_service`，再用 `-c test-ob-config.yml`

```bash
connectors -c /path/to/config-oceanbase.yml <子命令> ...
sync-ob -c /path/to/config-oceanbase.yml -i <connector_id>
```

## 四、CLI 命令速查

以下命令均需带 `-c <OceanBase 配置>`，使用安装后的 `connectors` 命令。

| 能力 | 命令示例 |
|------|----------|
| 看帮助 | `connectors --help` / `connectors -c config.yml --help` |
| 列出 connector | `connectors -c config.yml connector list` |
| 创建 connector | `connectors -c config.yml connector create --index-name <索引名> --service-type <类型> --name <名称> --index-language en --from-file <JSON>` |
| 列出索引 | `connectors -c config.yml index list` |
| 清空索引 | `connectors -c config.yml index clean <index_name>` |
| 删除索引 | `connectors -c config.yml index delete <index_name>` |
| 列出某 connector 的 job | `connectors -c config.yml job list <connector_id>` |
| 启动 job | `connectors -c config.yml job start -i <connector_id> -t full`（或 `incremental` / `access_control`） |
| 查看 job | `connectors -c config.yml job view <job_id>` |
| 取消 job | `connectors -c config.yml job cancel <job_id>` |
| 一次性同步到 OceanBase | `sync-ob -c config.yml -i <connector_id>`（独立命令，非 connectors 子命令） |

**关于 connector_id**：创建 connector 时由系统生成并打印在输出里（如 `Connector ... ID: 50064af25b174ed9abec ... has been created!`）。`job list`、`job start`、`sync-ob` 等必须指定是哪个 connector，**不能省略**。若不知道 ID，先执行 `connectors -c config.yml connector list` 查看已有 connector 及其 ID。

**说明**：`login` 用于连接 Elasticsearch，纯 OceanBase 时不需要。

## 五、典型流程（数据迁到 OceanBase）

1. **安装**：按第二节先装 SDK 再装 whl。  
2. **准备配置**：写好 OceanBase 的 YAML 与数据源 JSON（见 [04-数据源操作指南](./04-数据源操作指南.md)、[05-示例配置汇总](./05-示例配置汇总.md)）。  
3. **创建 connector**：`connectors -c config.yml connector create ... --from-file <数据源.json>`，记下输出的 **Connector ID**。  
4. **启动任务**：`connectors -c config.yml job start -i <connector_id> -t full`。  
5. **执行同步**：`sync-ob -c config.yml -i <connector_id>`，将数据写入 OceanBase 内容表。

## 六、最小验证流程

在任意目录执行（将 `config.yml` 换成你的配置文件路径；注意 `-c` 为相对当前目录，见 3.2）：

```bash
# 1. CLI 与连接
connectors --help
connectors -c config.yml connector list
connectors -c config.yml index list

# 2. 若有 connector，可测 job
connectors -c config.yml job list <connector_id>
connectors -c config.yml job start -i <connector_id> -t full
# 预期输出："The job <job_id> has been started."

# 3. 一次性同步到 OceanBase
sync-ob -c config.yml -i <connector_id>

# 4. 验证 OB 表与行数（若脚本已安装）
python -m scripts.verify_ob_connector -c config.yml
```

## 七、创建 connector 的通用模板

非交互创建需用 `--from-file` 传入 JSON，键名与各数据源配置一致（见 [04-数据源操作指南](./04-数据源操作指南.md)）：

```bash
connectors -c config.yml connector create \
  --name <显示名称> \
  --index-name <索引名> \
  --service-type <mysql|github|mongodb|...> \
  --index-language en \
  --from-file <path-to-config.json>
```

创建成功会输出 **Connector ID**；OceanBase 模式下可能提示 “Cannot create a connector-specific API key...”，可忽略。
