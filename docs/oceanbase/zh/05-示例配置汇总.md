# 各数据源示例配置汇总

以下为各 `service_type` 在 **非交互创建 connector**（`--from-file <JSON>`）时的示例 JSON。请将占位符替换为实际值，并按需增删键。具体必填项与依赖关系以各源 `get_default_configuration()` 为准（见 `connectors/sources/<名称>/datasource.py`）。

---

## mysql

```json
{
  "host": "127.0.0.1",
  "port": 3306,
  "user": "root",
  "password": "changeme",
  "database": "test",
  "tables": "*",
  "ssl_enabled": false,
  "fetch_size": 50,
  "retry_count": 3
}
```

## github

```json
{
  "data_source": "github_cloud",
  "auth_method": "personal_access_token",
  "token": "ghp_你的PAT",
  "repo_type": "other",
  "repositories": ["owner/repo"],
  "ssl_enabled": false,
  "retry_count": 3,
  "use_text_extraction_service": false,
  "use_document_level_security": false
}
```

GitHub Server 时增加：`"data_source": "github_server"`、`"host": "https://your-ghe-host"`。PAT 需具备 `repo`、`user`、`read:org`；推荐 Classic PAT（`ghp_`）。

## 使用说明

1. 将上述任一 JSON 保存为文件（如 `mysql-config.json`）。
2. 创建 connector 时指定该文件与对应 `service_type`：
   ```bash
   connectors -c config.yml connector create \
     --name my-conn \
     --index-name my-idx \
     --service-type <上表类型> \
     --index-language en \
     --from-file mysql-config.json
   ```
3. 若 CLI 提示“某键未在 JSON 中找到”，说明该数据源在非交互模式下要求该键，请在 JSON 中补全（可对照对应 `datasource.py` 中的 `get_default_configuration()`）。
