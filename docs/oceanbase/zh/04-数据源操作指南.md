**每个数据源的完整示例 JSON 见 → [05-示例配置汇总](./05-示例配置汇总.md)**（可复制后修改使用）。

## MySQL

- **service_type**：`mysql`
- **用途**：从 MySQL 库表同步数据到 OceanBase 内容表。

### 配置项（--from-file JSON 键）

| 键 | 说明 |
|----|------|
| host | 主机 |
| port | 端口（数字） |
| user | 用户名 |
| password | 密码 |
| database | 库名 |
| tables | 表列表，如 `"*"` 或 `["t1","t2"]` |
| ssl_enabled | 是否 SSL（bool） |
| fetch_size | 每批行数（可选） |
| retry_count | 重试次数（可选） |

### JSON 示例

详见 [05-示例配置汇总](./05-示例配置汇总.md#mysql)。最小示例：

```json
{
  "host": "127.0.0.1",
  "port": 3306,
  "user": "root",
  "password": "changeme",
  "database": "test",
  "tables": "*",
  "ssl_enabled": false,
  "fetch_size": 50,
  "retry_count": 3
}
```

### 命令示例

```bash
connectors -c config.yml connector create \
  --name ob-mysql-test \
  --index-name ob-mysql-idx \
  --service-type mysql \
  --index-language en \
  --from-file mysql-connector-config.json
```

## GitHub

- **service_type**：`github`
- **用途**：同步 GitHub 仓库（issues、PR、文件等）到 OceanBase 内容表。
- **凭证**：推荐使用 **Classic PAT**（`ghp_` 开头），需权限 `repo`、`user`、`read:org`。Fine-grained PAT（`github_pat_`）当前校验逻辑不支持，会报权限错误。

### 配置项（--from-file JSON 键）

| 键 | 说明 |
|----|------|
| data_source | `github_cloud` 或 `github_server` |
| auth_method | `personal_access_token` 或 `github_app` |
| token | PAT 或 App 凭证 |
| repo_type | `organization` 或 `other` |
| repositories | 仓库列表，如 `["owner/repo"]` 或 `["*"]` |
| ssl_enabled | bool |
| retry_count | 数字（必填，如 3） |
| use_text_extraction_service | bool（必填，如 false） |
| use_document_level_security | bool（必填，如 false） |
| host | 仅当 data_source=github_server 时需填 |

### JSON 示例

详见 [05-示例配置汇总](./05-示例配置汇总.md#github)。示例文件：`app/connectors_service/github-connector-config.example.json`，复制后改 `token` 和 `repositories`。

### 操作流程

1. 创建 connector：`connector create --service-type github --from-file github-connector-config.json ...`
2. 创建 job：`job start -i <connector_id> -t full`
3. 执行同步：`sync-ob -c <config> -i <connector_id>`

## 其他数据源

上表「数据源一览」中未单独列出的类型（如 azure_blob_storage、box、confluence、dropbox、gmail、gitlab、graphql、jira、notion、redis、salesforce、sandfly、servicenow、slack、zoom 等）操作方式一致：

1. **配置**：为该数据源准备 JSON 文件。**各数据源示例 JSON 见 [05-示例配置汇总](./05-示例配置汇总.md)**（redis、graphql、slack、gitlab、notion、jira、confluence、zoom、salesforce、servicenow、azure_blob_storage、box、dropbox、network_drive、sandfly 等均有示例）。键名与对应源码中 `get_default_configuration()` 一致；非交互创建时 JSON 需包含必填键及 CLI 依赖键。
2. **创建 connector**：
   ```bash
   connectors -c <OceanBase配置> connector create \
     --name <名称> \
     --index-name <索引名> \
     --service-type <上表中的 service_type> \
     --index-language en \
     --from-file <config.json>
   ```
3. **创建 job**：`job start -i <connector_id> -t full`（或 incremental / access_control，视数据源支持）。
4. **执行同步**：`sync-ob -c <config> -i <connector_id>`，或使用常驻 Service。

部分数据源有同步规则文档：如 [sync-rules/SALESFORCE.md](../../sync-rules/SALESFORCE.md)、[sync-rules/NOTION.md](../../sync-rules/NOTION.md) 等，可按需查阅。
