# 常见问题与脚本（OceanBase 后端）

FAQ（含配置文件路径、token 权限等）、让内容表有数据的两种方式，以及 verify_ob_connector / sync-ob 用法。

---

## 一、常见问题

1. **“Cannot create a connector-specific API key...”**  
   OceanBase 模式下不会创建 ES API Key，此提示可忽略。

2. **找不到配置文件 / `-c`: No such file or directory`**  
   `-c` 的路径是**相对当前工作目录**的。若在仓库根目录执行而配置在 `app/connectors_service/`，请写 `connectors -c app/connectors_service/配置文件名 ...`，或先 `cd app/connectors_service` 再用 `-c 配置文件名`。也可使用绝对路径。

3. **aiomysql 报错**  
   安装：`pip install aiomysql`，并确认 OceanBase 使用 MySQL 协议端口。

4. **connector create 交互太多**  
   使用 `--from-file <JSON>`，JSON 中包含该 service-type 所需全部键（见 [04-数据源操作指南](./04-数据源操作指南.md)），可完全非交互创建。

5. **内容表为何是 0 条？**  
   CLI 只负责在 OceanBase 里建 connector、job 记录和内容表结构；**实际把数据源数据写入内容表的是 Connector Service 或一次性同步脚本**。见下文「让内容表有数据」。

6. **job start 没有输出 “The job xxx has been started.”**  
   若已修复代码中的 `if job_id` 分支，应能正常输出；否则检查 `job start` 是否返回了有效的 job_id。

## 二、让内容表有数据：两种方式

### 方式 A：一次性同步（推荐先试）

安装 whl 后，在任意目录执行（需先有一个 pending 的 full 任务）：

```bash
sync-ob -c config.yml -i <connector_id>
```

若没有 pending job，先用 CLI 创建：

```bash
connectors -c config.yml job start -i <connector_id> -t full
```

再执行 `sync-ob`。会从数据源拉取数据并写入 OceanBase 的 `ob_content_<index_name_sanitized>` 表。

**可选参数**：`-j <job_id>` 指定要执行的 job，不指定则取该 connector 的第一个 pending full/incremental job。

### 方式 B：常驻 Connector Service

配置中需包含 `backend: oceanbase`、`oceanbase`、`connectors`（列出 connector_id 与 service_type），以及框架默认合并的 `service`/`sources`。然后启动：

```bash
elastic-ingest -c /path/to/config.yml
```

Service 会轮询 OceanBase 中的 pending job 并执行同步，结果写入 OceanBase 内容表。

## 三、verify_ob_connector（验证 OB 状态）

用于连接 OceanBase，列出系统表、内容表及行数，便于确认后端状态。安装 whl 后可用模块方式运行：

```bash
# 使用配置文件（推荐）
python -m scripts.verify_ob_connector -c config.yml

# 使用环境变量
OB_HOST=127.0.0.1 OB_PORT=2881 OB_USER=root@test OB_PASSWORD=xxx OB_DB=test \
  python -m scripts.verify_ob_connector
```

**输出示例**：当前库中的表列表、`ob_connectors` / `ob_connectors_sync_jobs` 行数及样例、各 `ob_content_*` 表的文档数。

## 四、sync-ob（一次性同步）

对指定 connector 执行一次 pending 的 full/incremental 同步，将数据写入 OceanBase 内容表。**独立命令**（不是 `connectors` 的子命令）。

**用法**：

```bash
sync-ob -c <config.yml> -i <connector_id> [-j <job_id>]
```

| 参数 | 说明 |
|------|------|
| `-c` / `--config` | 必填。含 `backend: oceanbase` 和 `oceanbase` 的 YAML 配置路径 |
| `-i` / `--connector-id` | 必填。Connector ID |
| `-j` / `--job-id` | 可选。指定要执行的 job；不指定则取该 connector 的第一个 pending full/incremental job |

**依赖**：配置中需有 `backend: oceanbase`、`oceanbase`；该 connector 在 OceanBase 中已存在，且有一个 pending 的 full 或 incremental job（可用 `connectors -c config.yml job start -i <connector_id> -t full` 创建）。

| 模块 | 优化内容 |
|------|----------|

| **ob/client.py** | **bulk_insert**：按表聚合后用 `executemany` 批量 REPLACE/DELETE；**yield_existing_documents_metadata**：使用 `DictCursor` 正确解析行 |
