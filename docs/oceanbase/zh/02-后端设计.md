# OceanBase 后端设计

OceanBase 作为 Connectors 后端时，用 MySQL 协议库下的若干表模拟 Elasticsearch 的索引与文档模型，便于 CLI 与 Connector Service 读写元数据与同步内容。使用方式与命令见 [01-快速开始](./01-快速开始.md)。

---

## 一、表分类

| 类型 | 表名 | 说明 |
|------|------|------|
| 系统表 | `ob_connectors` | Connector 元数据（对应 ES `.elastic-connectors-v1`） |
| 系统表 | `ob_connectors_sync_jobs` | 同步任务元数据（对应 ES `.elastic-connectors-sync-jobs-v1`） |
| 内容表 | `ob_content_<sanitized_index_name>` | 各索引的文档数据，一张表对应一个逻辑索引 |

---

## 二、命名规则

- **索引名 → 内容表名**：索引名中非字母数字的字符替换为下划线，再加前缀 `ob_content_`。
- 示例：
  - `github-idx` → `ob_content_github_idx`
  - `ob-mysql-idx` → `ob_content_ob_mysql_idx`
  - `my_index` → `ob_content_my_index`

---

## 三、表结构（DDL）

### 3.1 系统表 `ob_connectors`

```sql
CREATE TABLE IF NOT EXISTS ob_connectors (
    id VARCHAR(512) NOT NULL PRIMARY KEY COMMENT 'connector id',
    doc JSON NOT NULL COMMENT 'connector document body'
);
```

- `id`：Connector ID（如 CLI 返回的 20 位字符串）。
- `doc`：整条 connector 文档（JSON），含 configuration、index_name、service_type 等。

### 3.2 系统表 `ob_connectors_sync_jobs`

```sql
CREATE TABLE IF NOT EXISTS ob_connectors_sync_jobs (
    id VARCHAR(512) NOT NULL PRIMARY KEY COMMENT 'sync job id',
    doc JSON NOT NULL COMMENT 'sync job document body'
);
```

- `id`：Sync Job ID。
- `doc`：整条 job 文档（JSON），含 status、job_type、connector、indexed_document_count 等。

### 3.3 内容表 `ob_content_<name>`

```sql
CREATE TABLE IF NOT EXISTS `ob_content_<name>` (
    id VARCHAR(512) NOT NULL PRIMARY KEY COMMENT 'document _id',
    doc JSON NOT NULL COMMENT 'document _source',
    _timestamp TIMESTAMP NULL DEFAULT NULL COMMENT 'for incremental sync'
);
```

- `id`：文档 `_id`。
- `doc`：文档内容（`_source`）的 JSON。
- `_timestamp`：用于增量同步的时间戳（可为 NULL）。

---

## 四、与 Elasticsearch 的对应关系

| ES 概念 | OceanBase 对应 |
|---------|-----------------|
| 系统索引 `.elastic-connectors-v1` | 表 `ob_connectors` |
| 系统索引 `.elastic-connectors-sync-jobs-v1` | 表 `ob_connectors_sync_jobs` |
| 普通索引（如 `my-index`） | 表 `ob_content_my_index` |
| 文档 `_id` | 列 `id` |
| 文档 `_source` | 列 `doc`（JSON） |
